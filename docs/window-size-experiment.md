# 窗口大小实验：LLM 空闲分类在不同有效行数下的表现

## 实验背景

看门狗检测到会话空闲后，需要分类空闲原因（等待决策 / 任务完成 / 不明）。
分类依赖 tmux 输出的截取窗口。窗口太小可能漏掉关键上下文，太大可能引入历史干扰。

## 实验设计

### 采样方式
**只在空闲边界点采样**：找到 tmux 输出中最后一个 `❯` 提示符位置，取该边界之前的 N 行有效内容。
不是随机采样，只在会话从执行中转为空闲的边界点才有分类价值。

### 有效行定义
从 tmux 原始输出中，先经过 `_strip_noise()` 过滤以下噪音行，剩余为有效行：
- 纯分隔线（`────`、`═══`、`─────` 等）
- Box drawing 字符行（`┌─┐`、`╔═╗` 等）
- 空行和纯空格行
- 纯 `❯` 提示符行
- 状态栏行（`模型:`、`输入:`、`会话:`、`目录:`）
- `⏵⏵` 状态指示行
- 长分隔线（`─{10,}`）

### 窗口大小
6 组：30 / 50 / 80 / 100 / 150 / 200 有效行

### 分类方式
全部使用 LLM（MiniMax M2.7-highspeed）分类，提示词要求判断 Claude 最后一段输出的状态。

### 样本
覆盖所有在线 tmux 会话中有足够内容的空闲边界点，共 11 个会话。

## 实验结果

| 会话 | 30行 | 50行 | 80行 | 100行 | 150行 | 200行 | 稳定区间 |
|------|------|------|------|-------|-------|-------|----------|
| auto_finance | D | D | D | D | D | D | 全部 |
| benchmark-tmp | D | D | D | D | D | D | 全部 |
| cli | C | C | C | C | C | C | 全部 |
| **context** | **D** | **D** | **D** | **D** | **C** | **C** | 30-100 |
| gps | C | C | C | C | C | C | 全部 |
| html | C | C | C | **?** | C | C | 30-80 |
| kwcode | C | C | C | C | C | C | 全部 |
| multi_stats | C | C | C | C | C | C | 全部 |
| video | D | D | D | D | D | D | 全部 |
| **watchdog** | **D** | **?** | **D** | **C** | **C** | **C** | 30-80 |
| watchdog-dup | C | C | C | C | C | C | 全部 |

D = decision_needed（等待人工决策）
C = task_complete（任务完成，等待验收）
? = idle_unknown（无法判断）

## 关键发现

### 1. 50 行有效内容是安全上限
所有 11 个会话在 50 行窗口下分类结果均正确。没有因窗口不足导致漏判的情况。

### 2. 超过 80-100 行会被历史干扰
- `context`：30-100 行正确判定为 decision_needed，150/200 行被更早的完成状态覆盖，错误判为 task_complete
- `watchdog`：30-80 行正确判定为 decision_needed，100 行以后被历史干扰变成 task_complete
- `html`：30-80 行正确判定为 task_complete，100 行时被干扰判为 unknown

原因：大窗口包含多个交互回合（前面"做完了"→ 中间"还有问题"→ 最后"卡住了"），LLM 被前面的内容带偏。

### 3. 30 行有轻微风险但本次实验未触发
30 行对当前 11 个会话都够用，但考虑到一次完整的 Claude 交互（输出代码 → diff → commit → 提问）通常需要 20-40 行，30 行可能在某些场景下刚好截断关键的决策问题。

### 4. 提示词是关键变量
窗口大小的影响远小于提示词措辞的影响。通过提示词明确要求"只看最后一段输出"可以缓解大窗口的干扰问题。

## 对照数据：关键字 vs LLM

| 会话 | 关键字(50行) | LLM(50行) | 差异 |
|------|-------------|-----------|------|
| auto_finance | unknown | decision_needed | 关键字漏判 |
| gps | unknown | task_complete | 关键字漏判 |
| html | unknown | task_complete | 关键字漏判 |
| kwcode | unknown | task_complete | 关键字漏判 |

关键字匹配对 8 个会话中 4 个判为 unknown，LLM 全部正确分类。说明当前关键字覆盖面不足，建议 unknown 时也调 LLM。

## 决策建议

- **窗口大小**：50 行有效内容，原始抓取 300 行
- **分类策略**：关键字先行过滤明确的 decision/complete，unknown 的调 LLM 二次分类
- **提示词优化**：明确要求 LLM 只关注最后一段输出，忽略历史交互
